## Morning — DB Lock from Zombie Process

### What the orchestrator sent to Discord at 5:03 AM / 6:01 AM

```
[NBA] NBA GRADING - 2026-03-03

[NBA] NBA PREDICTIONS - 2026-03-04
Generated: 0 predictions   Probability Variety: 0 unique values
```

### Root cause

**Grading (5:03 AM) — expected, not a bug.**
Mar 3 never had predictions generated (the scipy hang from yesterday's session meant the
prediction script ran at 12:01 PM, those rows were deleted when the session started, and
regeneration never completed). No predictions → nothing to grade → `0/0`. Correct behavior.

**Predictions (6:01 AM) — `sqlite3.OperationalError: database is locked`.**
Yesterday's manual `generate_predictions_daily_V6.py 2026-03-03 --force` command was still
running as a zombie process (PID 118688). It had been alive for ~9 hours, having hung on the
Windows Defender scipy block from yesterday's session. It held a write lock on
`nba_predictions.db`, preventing the orchestrator's 6:01 AM prediction run from writing.
The orchestrator's script exited with code 1 and reported 0 predictions.

### Fix

Killed PID 118688, then ran manually:

```bash
python -u nba/scripts/generate_predictions_daily_V6.py 2026-03-04 --force
python sync/supabase_sync.py --sport nba --date 2026-03-04
```

### Result

- **1,047 NBA predictions** generated for Mar 4 (6 games, 54 PP-matched players)
- **518 smart picks** synced to Supabase
- **512 odds_type** labels corrected, **1,211 rows** with game times populated
- Tier distribution in Supabase: T1=373, T2=101, T3=116, T4=297, T5=113

Sklearn `UserWarning: X does not have valid feature names` appeared throughout output —
harmless (feature array vs named DataFrame mismatch at predict time), does not affect
prediction values. Suppress on next retrain by passing a DataFrame instead of numpy array
to the model's `predict_proba()`.

## Evening — Dashboard Pick Display Review

### Observation: NBA shows 189 picks, all T1-ELITE; NHL shows T1–T3 spread

**The 189 vs 518:** `fetch_picks()` has a hard `.limit(200)` ordered by `ai_probability`
DESC, plus default filters in the "More filters" expander (collapsed by default):
- min_prob = 56%
- min_edge = 5%
- tier_filter = [T1-ELITE, T2-STRONG, T3-GOOD]

These defaults reduce 518 synced smart picks to the top 189 high-confidence picks.

**All T1-ELITE:** Even with all filters cleared (min_prob=50%, min_edge=0%), the visible
NBA picks still all show as T1-ELITE. Supabase has the full tier spread in the data,
but the NBA ML models (79–99% accuracy) produce high-confidence probabilities for their
positive picks, which cluster in T1-ELITE (≥75%) territory. This is different from NHL
where the model produces a wider probability spread. This behavior is noted as a TODO —
see `docs/github_issues/nba-tier-distribution.md`.

**User verdict:** Fine with T1-only NBA picks for now. The data is correct.
