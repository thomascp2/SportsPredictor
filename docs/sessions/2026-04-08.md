### B ‚Äî Golf Smart Picks: 3-Layer PrizePicks Bug Fix

Golf had 1,516 predictions (Valero TX Open) but 0 smart picks ever written. Root cause was
a chain of three bugs preventing PP golf lines from ever being fetched or matched.

**Bug 1 ‚Äî `shared/prizepicks_client.py`**
`get_lines_for_sport('GOLF')` filtered `league.upper() == 'GOLF'` but PrizePicks stores
golf under league name `'PGA'`. Zero lines were ever saved to the PP database for golf.
**Fix:** Added `SPORT_LEAGUE_ALIASES = {'GOLF': {'GOLF', 'PGA'}}` ‚Äî accepts either name.

**Bug 2 ‚Äî `shared/smart_pick_selector.py` (`_get_pp_lines`)**
No `GOLF` branch in the sport-specific props selector ‚Äî fell through to the NBA `else`
clause and queried NBA prop names (points, rebounds, etc.) against golf lines.
**Fix:** Added `elif self.sport == 'GOLF': props = ('round_score', 'make_cut')`

**Bug 3 ‚Äî `shared/smart_pick_selector.py` (`_get_pp_lines`)**
DB query used `league = self.sport` = `'GOLF'`, but saved rows have `league = 'PGA'`
(from PrizePicks API response). Query always returned 0 rows.
**Fix:** `league_name = 'PGA' if self.sport == 'GOLF' else self.sport`

**Discord webhook:** Added `GOLF_DISCORD_WEBHOOK` to `start_orchestrator.bat`.
