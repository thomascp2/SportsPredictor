### Root Causes Found & Fixed

#### Bug 1 — ESPN Boxscore: Only 5 Players Per Team Returned
**File:** `nba/scripts/espn_nba_api.py`
**Introduced by:** Self-healing auto-fix, December 2024
**Impact:** Grading rate ~23-31% instead of ~70%+

The auto-fix added an "enhanced DNP check":
```python
# WRONG — was filtering out bench players
if (athlete_data.get('didNotPlay', False) or
    athlete_data.get('inactive', False) or
    athlete_data.get('active', True) is False or      # ESPN sets active=False for ALL players post-game
    athlete_data.get('starter', False) is None):       # ESPN sets starter=null for ALL bench players
    continue
```

Verified via raw ESPN API inspection: for a completed game (ORL @ PHX 2026-02-21),
13 ORL athletes and 14 PHX athletes appear in the response. Players like Wendell
Carter Jr. and Dillon Brooks show `active: False, starter: True, didNotPlay: False` —
they started and played but were being filtered out. ESPN's `active` field means
"actively playing right now" (false once game ends), NOT "played in this game".

**Fix:** Only skip `didNotPlay: True`. Result: 10 players/game → ~22 players/game,
6 games × ~22 = 133 players (was 60).

#### Bug 2 — Grader Missing 7 Prop Types
**File:** `nba/scripts/auto_grade_multi_api_FIXED.py`

V6 predictions generate 13 prop types. The grader's `stat_map` only had 6:
`points, rebounds, assists, threes, stocks, pra, minutes`

Missing: `pts_rebs, pts_asts, rebs_asts, steals, blocked_shots, turnovers, fantasy`

For any unknown prop, `stat_map.get(prop_type)` returned `None`, and
`stats.get(None, 0)` returned `0`. So every OVER prediction for these props graded
as MISS (0 < any line), every UNDER graded as HIT. This silently corrupted grading
accuracy metrics.

**Fix:** Added all 7 missing prop types. Fantasy uses PrizePicks formula:
`PTS + 1.2*REB + 1.5*AST + 2.0*STL + 2.0*BLK - TOV`. Unknown prop types now
return `None` and are logged as ungraded rather than graded against 0.

---

#### Bug 3 — NBAStatsAPI Crashes on Unhandled urllib3 Exception
**File:** `nba/scripts/data_fetchers/nba_stats_api.py`

`_make_request()` only caught `requests.exceptions.RequestException`. In Python 3.13
with urllib3 2.x, some low-level connection errors (e.g., `RemoteDisconnected`,
`ProtocolError`) can surface as plain `Exception` types not wrapped by requests.
The script exited with code 1, causing the orchestrator to record a failure.

**Fix:** Added `except Exception as e` fallback catch.

---

#### Bug 4 — NHL Schedule Fetch Exits 1 on Off Days
**File:** `nhl/scripts/fetch_game_schedule_FINAL.py`

`fetch_game_schedule()` returned `False` (→ `sys.exit(1)`) when no games were
found on a given date. The orchestrator treated this as a pipeline error and added
it to `errors[]`, making every off-day run show `Result: FAILED` with no visible
error message (the error was in the list but only prediction errors are printed).

```python
# BEFORE
if not games:
    return False  # → exit 1 → orchestrator marks FAILED

# AFTER
if not games:
    print(f'[OK] No NHL games scheduled for {target_date} (off day or break)')
    return True  # → exit 0 → orchestrator marks SUCCESS
```

#### Bug 5 — NHL Grading Discord Wrong Signature + Off-Day Failure
**File:** `nhl/scripts/v2_auto_grade_yesterday_v3_RELIABLE.py`

Two issues:
1. `send_discord_notification(f"❌ GRADING FAILED\n{error_msg}")` — called with 1 arg,
   but function signature is `(title: str, message: str, color: str = "blue", ...)`.
   This raised `TypeError`, crashing the error handler itself.
2. "No predictions to grade" (legitimate on off days) was treated as a failure and
   triggered the broken Discord call. Changed to return 0 with a clean OK message.

Also fixed 3 other Discord calls throughout the file that had the same single-arg bug.

### What Was Built

#### 1 — Daily Top 20 Picks Discord Notification
**File:** `orchestrator.py`

Added `run_top_picks_notification()` method scheduled at 10:15 CST for both sports
(15 minutes after predictions generate). Queries today's predictions, selects the
highest-confidence pick per player, ranks by probability then L5/consistency
tiebreakers, posts to Discord webhook.

**Format:**
```
 1. Player Name (TEAM vs OPP)  OVER X.X PROP — 91% model | L5: 80% 4x streak
```

**Iteration history:**
- v1: Used `ABS(probability - 0.5)` as sort key → all 80% in learning mode, arbitrary order
- v2: Added `f_l5_success_rate, f_consistency_score, f_current_streak` tiebreakers; dropped `f_insufficient_data=1` and `f_games_played < 5` rows
- v3 (current): Confirmed L5% is always a multiple of 20% (it's hits/5 games); display is directional (OVER pick shows OVER rate, UNDER pick shows UNDER rate); example fixed

**Known data timing issue identified:** `f_l5_success_rate` and `f_current_streak` for
combo stats (PRA, fantasy, pts_rebs, etc.) can be 0 if the prediction pipeline runs
before the grading pipeline has updated `player_game_logs` for recent games.
Kevin Durant showed L5: 0% for PRA 29.5 despite going OVER in 5 of his last 5 games —
his PRA column in `player_game_logs` was NULL at prediction time. Normally grading
runs before predictions (scheduled earlier in the day), so this should be rare on
steady-state runs but will occur after breaks/restarts.

#### 4 — NBA V6 Wired to ProductionPredictor
**File:** `nba/scripts/generate_predictions_daily_V6.py`

NHL V6 already used `ProductionPredictor`. Added the same pattern to NBA V6:
- Statistical predictor always runs first (provides features + acts as fallback)
- ML ensemble applied when a trained model exists for the prop/line combo
- Falls back to statistical for niche/new prop types with no trained model

```python
stat_result = self.predictor.predict_prop(...)
if stat_result and self.ml_predictor and self.ml_predictor.is_model_available('nba', prop_type, line):
    result = self.ml_predictor.predict_ensemble('nba', prop_type, line,
                 stat_result.get('features', {}), stat_result)
else:
    result = stat_result
```
