## Issues Investigated & Fixed

### Bug 1 ‚Äî Discord Top-20 Picks Never Firing

**Symptom:** No Discord messages being received for either sport despite the orchestrator
running continuously.

**Root cause:** `DISCORD_WEBHOOK_URL` env var was never set. The orchestrator reads it
from `os.getenv('DISCORD_WEBHOOK_URL', '')` at startup ‚Äî if empty, `run_top_picks_notification()`
immediately returns with "[SKIP] Top picks: Discord webhook not configured". The `.env` file
only contained Supabase credentials and had no Discord webhook. The orchestrator also has no
`load_dotenv()` call, so `.env` is never loaded anyway. The `start_orchestrator.bat` set no
env vars at all.

**Fix (`start_orchestrator.bat`):**
- Added `SET DISCORD_WEBHOOK_URL=<webhook>` before the python call
- Added `SET NHL_DISCORD_WEBHOOK=<webhook>` (sport-specific, used by post-prediction path)
- Added `SET NBA_DISCORD_WEBHOOK=<webhook>` (same)
- All three currently point to the same webhook (single channel). Can route to separate
  channels later without any code changes ‚Äî just swap the URLs in the bat file.

**Verified:** Manually triggered `run_top_picks_notification()` for both sports via inline
Python. All three messages posted successfully (NHL TOP 20 PICKS, NHL TOP 20 OVERS, NBA TOP 20).

### Feature ‚Äî `!parlay` Command Added to Discord Bot

**Request:** User wanted a quick on-the-go command to generate a quality parlay from
today's top model picks.

**Design decisions:**
- Samples randomly from the **top 10** picks per sport (not all picks) ‚Äî ensures every
  leg comes from the model's highest-confidence predictions while giving variety per call.
- Supports `!parlay [nba|nhl|both] [2-6]` syntax. Default: `!parlay` ‚Üí 4-leg, both sports.
- Shows combined probability so users understand parlay risk.
- Prints "Run !parlay again for a different combo" ‚Äî makes it clear re-rolling is supported.

**Changes (`discord_bot.py`):**
- Added `import json` and `import random`
- Added `_fetch_top_picks_for_parlay(sport, game_date, pool_size)` ‚Äî mirrors
  `orchestrator._fetch_top_picks()` SQL logic directly in the bot (no cross-import needed).
  NBA uses `f_*` column query; NHL parses `features_json` in Python.
- Added `_format_parlay_leg(i, pick)` ‚Äî compact single-line formatter with sport tag.
- Added `@bot.command(name='parlay')` ‚Äî validates args, builds pool, samples legs,
  computes combined probability, posts formatted message.
- Updated docstring at top of file to document the new command.

### Bug 2 ‚Äî `!picks nhl` Only Returning 2-3 Picks

**Symptom:** `!picks nhl` returned only 2 picks; `!picks nba` returned 208. With 360
shots predictions and 213 points predictions in the DB for today, something was filtering
almost everything out.

**Root cause:** `SmartPickSelector._get_pp_lines()` fetched 285 PrizePicks NHL lines.
`_get_predictions_with_params()` fetched 573 NHL predictions. The player lookup builds
keys as `(player_name.lower(), prop_type)` from both sides and tries to match them.

The mismatch: **our NHL prediction DB stores abbreviated names** (`A. Fox`, `S. Crosby`,
`E. Malkin`) while **PrizePicks uses full names** (`Adam Fox`, `Sidney Crosby`,
`Evgeni Malkin`). The key `('adam fox', 'shots')` never matched `('a. fox', 'shots')`.

The fallback fuzzy matcher (fuzzywuzzy `fuzz.ratio`) requires `>= 85` similarity.
`fuzz.ratio('adam fox', 'a. fox')` scores approximately 65 ‚Äî fails for virtually every
NHL player. Only 3 picks with accidentally matching names slipped through.

**Fix (`shared/smart_pick_selector.py`):**

Added `_is_initial_match(full_name, abbrev_name)` method:
```python
def _is_initial_match(self, full_name: str, abbrev_name: str) -> bool:
    # Matches 'a. fox' against 'adam fox':
    # - parse initial ('a') and last name ('fox') from abbrev_name
    # - compare initial to first letter of full_name's first word
    # - compare last name to full_name's last word
```

In the player lookup loop, added initial-match check **before** the fuzzy fallback:
```python
if self._is_initial_match(pp_name_lower, pred_name):
    best_match_key = pred_key
    best_match_score = 100
    break  # confident match, no need to continue
# fall through to fuzz.ratio for other cases
```

**Result:** NHL `!picks` went from **3 picks ‚Üí 39 picks** (matched 189 of 285 PP lines).
The unmatched 96 lines are players with no prediction for today (bench players,
last-minute scratches, etc.) ‚Äî expected and correct behavior.

**NBA unaffected:** NBA prediction DB stores full player names, so the existing exact/fuzzy
match already worked correctly (208 picks before and after).
