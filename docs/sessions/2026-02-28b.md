## Option A тАФ Cloudflare Tunnel

**Goal:** Get the dashboard accessible on phone immediately, zero code changes.

**Steps:**
1. `winget` not available in Git Bash environment.
2. Downloaded `cloudflared.exe` directly from GitHub releases:
   ```
   curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-windows-amd64.exe -o C:\Users\thoma\cloudflared.exe
   ```
3. Ran quick tunnel (no Cloudflare account needed for ad-hoc URLs):
   ```
   cloudflared.exe tunnel --url http://localhost:8502
   ```
4. Tunnel URL generated: `https://bidding-transaction-clinton-dare.trycloudflare.com`
   *(Note: quick tunnel URLs change on each restart тАФ see Option B for a permanent URL)*

**New file (`start_dashboard.bat`):**
- Starts Streamlit on port 8502 (`cloud_dashboard.py`) in a sub-window
- Waits 5 seconds for Streamlit to boot, then starts the Cloudflare tunnel
- Auto-restarts tunnel with 10s cooldown if it drops
- Gitignored (contains Supabase credentials)

**`start_orchestrator.bat` update:**
- Added `start "FreePicks Dashboard" start_dashboard.bat` тАФ dashboard + tunnel now
  launch automatically alongside the orchestrator and Discord bot on every startup

**Caveats:**
- Quick tunnel URL changes on every restart (no persistent URL without a Cloudflare account)
- Goes down when PC sleeps or shuts off тАФ Option B (Streamlit Cloud) solves both issues
- `cloudflared.exe` is stored at `C:\Users\thoma\cloudflared.exe` (not in repo)

## Option B тАФ Supabase-Backed Cloud Dashboard

**Goal:** A permanently-accessible, always-on dashboard that reads from Supabase (not
local SQLite) and can be deployed to Streamlit Community Cloud for free.

**Key insight:** Supabase already has 22,111 rows in `daily_props` with all the fields
needed (`ai_prediction`, `ai_probability`, `ai_edge`, `ai_tier`, `ai_ev_*`, etc.) тАФ
the sync layer has been keeping it current since Feb 14.

**New file (`dashboards/cloud_dashboard.py`):**

Three-tab dashboard reading entirely from Supabase:

| Tab | Contents |
|-----|----------|
| **Today's Picks** | Sport/date/direction/tier/prob/edge filters; picks table; by-prop expanders; interactive parlay builder with combined probability |
| **Performance** | 14-day accuracy (overall + by tier + by prop); model accuracy line chart from `model_performance` table |
| **System** | Pipeline status per sport (Today тЬЕ / Yesterday ЁЯЯб / Stale ЁЯФ┤); total props + graded counts; force-refresh button |

**Design decisions:**
- `initial_sidebar_state="collapsed"` тАФ better default for mobile (sidebar hidden until needed)
- Credentials: reads `st.secrets["SUPABASE_URL"]` and `st.secrets["SUPABASE_KEY"]`
  (Streamlit Cloud secrets UI) with env var fallback for local runs
- All queries cached with `@st.cache_data(ttl=300)` тАФ 5 min cache, force-refresh button available
- `@st.cache_resource` for the Supabase client singleton

**New file (`dashboards/requirements.txt`):**
```
streamlit>=1.32.0
supabase>=2.3.0
pandas>=2.0.0
```
Required by Streamlit Community Cloud to know what to install.

**New file (`.streamlit/secrets.toml`):**
- Contains Supabase URL + service role key for local Streamlit runs
- Gitignored тАФ never committed

**`.gitignore` additions:**
```
.streamlit/secrets.toml
start_dashboard.bat
```

**Verified:** Dashboard running at `http://localhost:8502`, HTTP 200. Opens in browser.

## Streamlit Community Cloud Deployment (Option B тАФ next step)

To make it always-on and PC-independent:

1. Push this commit to GitHub (`thomascp2/SportsPredictor`)
2. Go to **share.streamlit.io** тЖТ Sign in with GitHub
3. New app тЖТ select repo `thomascp2/SportsPredictor`
4. Set **Main file path**: `dashboards/cloud_dashboard.py`
5. Under **Advanced settings тЖТ Secrets**, add:
   ```toml
   SUPABASE_URL = "https://***REMOVED***"
   SUPABASE_KEY = "<service_role_key>"
   ```
6. Deploy тАФ get a permanent `https://yourapp.streamlit.app` URL

This gives a mobile-bookmarkable URL that works 24/7 regardless of whether the home PC is on.
