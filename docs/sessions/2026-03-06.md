## Issue Identified — ai_edge Stale After odds_type Relabeling

### What the user noticed

On the dashboard, two NHL players on identical lines (shots goblin OVER 1.5)
showed similar edges despite very different probabilities:

| Player | Probability | Edge (before fix) |
|--------|------------|-------------------|
| T. Hertl | 76.4% | +20.4% |
| Z. Hyman | 94.9% | +18.9% |

Intuitively wrong — if Hertl is barely above the goblin break-even and Hyman
is well above it, their edges should not be similar.

### Root Cause

`ai_edge` is calculated as `(ai_probability − break_even) × 100` where:

```
standard break-even = 0.56   (56%)
goblin   break-even = 0.76   (76%)
demon    break-even = 0.45   (45%)
```

The pipeline has three sync steps:

1. **`sync_predictions()`** — writes all rows with `odds_type = 'standard'` and
   `edge = (prob − 0.56) × 100`.
2. **`sync_smart_picks()`** — runs SmartPickSelector, which uses correct
   per-type break-evens and upserts goblin/demon rows with proper edges.
   Fails silently when player name can't be matched (e.g. "T. Hertl" in our
   DB vs "Tomas Hertl" on PrizePicks).
3. **`sync_odds_types()`** — label-only correction. Has better name matching
   (initial-name fallback for NHL). Updates `odds_type` to match PrizePicks,
   but previously **never touched `ai_edge`**.

So any player that fell through step 2 due to a name mismatch got their
`odds_type` correctly relabeled to 'goblin' in step 3, but kept the standard
break-even edge. Hertl: 76.4% − 56% = **20.4% shown**, true goblin edge only
0.4%. Hyman matched in step 2, so her edge was computed correctly: 18.9%.

### Fix — `sync/supabase_sync.py`

`sync_odds_types()` now:

1. Fetches `ai_probability` and `ai_prediction` in addition to the existing
   fields (needed for recalculation).
2. For every row, computes `expected_edge = (ai_probability − break_even[odds_type]) × 100`.
3. If the stored `ai_edge` differs from `expected_edge` by more than 0.05,
   patches the edge — regardless of whether the `odds_type` label changed.
   This catches both current-run relabelings AND rows relabeled in a prior
   sync run that still carry the stale edge.

### Backfill Applied (2026-03-06)

Ran `sync_odds_types()` for both sports immediately after the fix:

- **NHL: 312 rows corrected**
- **NBA: 1,179 rows corrected**

Hertl shots goblin edge corrected: **+20.4% → +0.4%** (barely break-even,
genuinely marginal pick). Hyman shots goblin unchanged at +18.9% (correct).

### Forward Behavior

`sync_odds_types()` is wired into the orchestrator and runs after every
prediction sync. All future days will be self-correcting automatically.
