# Session Notes — 2026-04-04

## Summary

Dashboard polish session. All changes in `dashboards/cloud_dashboard.py`.

## Changes Shipped

### Auto-refresh removed
- Removed `<meta http-equiv="refresh" content="60">` from the System tab.
- Was triggering a full browser reload every 60s, resetting tab state to Top Plays.

### Top Plays tab (new default home screen)
- Added as the first tab — Streamlit opens tab[0] on load, making it the default.
- Two-column layout: NHL | NBA side by side.
- Shows up to 6 picks per sport, T1-ELITE and T2-STRONG only (falls back to top-6 by edge if none qualify).
- Each card: player, matchup + time, prop/line/prediction, probability bar, edge %, tier badge, goblin/demon label.

### Game Lines tab
- Cards now sorted by game time earliest → latest.
- Game time displayed in card header (sourced from `prizepicks_lines.db` — same lookup `fetch_picks` uses, matched on home_team + date + league).
- **3 rows per card** instead of 6: one row per market (moneyline, spread, total), showing only the highest-probability side. The opposing side is dropped entirely.
- Progress bar renders only on the shown (favorable) row.

### Terminal / Bloomberg aesthetic
- Applied JetBrains Mono globally via Google Fonts `@import`.
- Targeted only text elements (inputs, buttons, labels, p, li, metrics) — structural `div`/`span` excluded to avoid breaking Streamlit widget layouts.
- Tabs: uppercase, letter-spaced, flat underline indicator (no rounded pill).
- Metrics: 10px uppercase labels, 20px bold values.
- Buttons: 4px radius, card background, blue hover border.
- Cards and badges: radius reduced from 10–12px → 4px (square terminal feel).
- Progress bars: thinner (5px), sharper radius (2px).

### Bug fixes

| Issue | Root cause | Fix |
|---|---|---|
| Dropdown text garbled (Today's Picks, MLB Season Props) | Global `line-height` on all `div`/`span` broke Streamlit's absolutely-positioned widget layers | Removed `div`, `span`, `[class*="css"]` from global font selector; removed global `line-height` override |
| h3 rule garbling dropdown labels | `h3 { text-transform: uppercase }` was hitting Streamlit internal widget headers | Scoped to `.stMarkdown h3` only |
| Dashboard title cut in half | `st.title()` container has fixed height that clips when mono font reflows | Replaced with `st.markdown()` div — no height constraint |
| MLB Season Props refresh button misaligned | Button floated to top of column; selectboxes offset by label height | Added 27px spacer before button |
| Date picker visually inconsistent with selectboxes | Different BaseWeb component, different default styling | Added targeted CSS for `[data-testid="stDateInput"]`: same font, size, bg, border, radius; hid calendar icon |
| Dashboard title invisible (CSS var not resolving) | `color: var(--text-primary)` doesn't resolve in Streamlit shadow DOM for heading elements | Switched to explicit hex `#e6edf3` |
| Game Lines game_time always None | `game_predictions` table has no `game_time` column | Post-query lookup from `prizepicks_lines.db` by home_team + date + league |
