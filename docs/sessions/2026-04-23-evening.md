# Session Handoff — 2026-04-23 Evening

## What Happened This Session

Continued data quality audit. Discovered the root cause of the "line duplication" inflation:
the V6 prediction generator stored ALL PrizePicks lines (standard + goblin + demon) as
predictions but wrote `odds_type='standard'` for every row. So goblin/demon line predictions
were miscounted as standard picks, corrupting profit math and win rates.

The `prizepicks_lines` table (Dec 2025 → present, 926K rows) IS the ground truth. Standard
lines average exactly **1.00–1.01 per player/prop/date** — one canonical line per day.

---

## What Was Committed This Session

### `tools/db_cleanup.py` — two new fix functions added (committed fd558cfb)

**`_fix_odds_type_labels(cursor, sport, dry_run)`**
- Joins `prediction_outcomes` to `prizepicks_lines` on `(player_name, game_date, prop_type, line)`
- Overwrites `odds_type` with the correct value from PrizePicks ground truth
- No heuristics — pure join. Must run BEFORE profit backfill and dedup.

**`_fix_line_duplication(cursor, sport, dry_run)`**
- After odds_type is corrected, tags remaining duplicate rows as `DUPLICATE_LINE`
- Prefers rows whose line exists in `prizepicks_lines`; median fallback for pre-Dec 2025 only
- Must run AFTER `_fix_odds_type_labels`

**Order in `clean_sport()`:**
```
impossible combos → DNP inflation → logic violations →
odds_type relabel → profit backfill → is_smart_pick sync → line dedup
```
