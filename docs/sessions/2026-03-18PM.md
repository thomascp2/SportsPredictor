## Issues Fixed

### 1. ESPN 3-Pointer Parsing Bug — Season-Long Data Corruption (CRITICAL)

**Root cause**: `nba/scripts/espn_nba_api.py` — `parse_made_attempted()` looked for the 3PM column under keys `['3PM-3PA', '3P', 'Three Pointers', '3FG', 'threePointers', '3PM/3PA']` but ESPN's actual column header is **`'3PT'`**. `FG` and `FT` worked because those short-form keys were already present. `3PT` was never added.

**How it was caught**: User noticed Isaiah Collier showing `threes_made = 0` across all games despite ESPN game log showing 0-5, 1-3, 2-5, etc. on multiple dates. DB query confirmed: every player, every game, threes = 0.

**Impact**: 3,742 of 6,623 threes prediction outcomes corrupted across 101 dates (2025-11-10 → 2026-03-17). Players who made 3-pointers were graded HIT on UNDER 0.5 picks they should have missed. Model signals (L5/L10 hit rates, streak, trend) for threes are unreliable for the entire season.

**Fix** (`nba/scripts/espn_nba_api.py` line 639):
```python
# Before
fg3m, fg3a = parse_made_attempted(['3PM-3PA', '3P', 'Three Pointers', '3FG', 'threePointers', '3PM/3PA'])
# After
fg3m, fg3a = parse_made_attempted(['3PM-3PA', '3PT', '3P', 'Three Pointers', '3FG', 'threePointers', '3PM/3PA'])
```

**Backfill**: `nba/scripts/backfill_threes_regrade.py` — iterates all 101 affected dates, re-runs grading script per date with corrected parsing, verifies improvement. Run overnight:
```bash
python nba/scripts/backfill_threes_regrade.py
```
After backfill, sync to Supabase (or `!refresh nba` via Discord).

**Prevention**: Consider a stat-level sanity check in grading — alert via Discord if >50% of players show 0 for any counting stat (would catch this class of parsing bug before data ships to model training).

### 2. Graded Outcome Display Inconsistency

**Symptom**: Nathan MacKinnon history showing OVER 1.5 Points → actual=1 → HIT (should be MISS).

**Root cause**: `mobile/src/services/api.ts` — `fetchPlayerHistory()` used `row.result` (Supabase `result` column) as authoritative outcome. The `result` column reflects the **model's original prediction** graded by the NHL script (e.g., UNDER 1.5 → HIT because 1 < 1.5). But `ai_prediction` in Supabase stores the **PP/smart pick direction** (OVER). When these diverge, display showed OVER 1.5 graded as HIT — factually inconsistent.

**Fix**: Always compute outcome from `actual_value + ai_prediction` directly:
```typescript
// Removed: if (row.result) { outcome = row.result; }
if (row.actual_value != null && row.ai_prediction) {
  outcome = (row.ai_prediction === 'OVER' ? row.actual_value > row.line : row.actual_value < row.line)
    ? 'HIT' : 'MISS';
}
```
Also removed `result` from Supabase SELECT — no longer needed.

### 3. Grading Sync Gap (3/15–3/17 missing from Supabase)

**Root cause**: `orchestrator.py` — Supabase sync was gated on `if success and SUPABASE_SYNC_AVAILABLE`. Any non-fatal grading error (a few players not found in box score) caused `success = False`, silently skipping the sync with no Discord notification. Mobile app showed stale data for multiple days.

**Fix**: Removed the `success` gate — sync always runs after grading (partial results are better than stale data). Added Discord alerts for two failure modes:
- Sync threw an exception (network, auth, etc.)
- Sync completed but pushed 0 rows (grading/match failure)

### 5. Prop Card Display Bugs (earlier in session)

**GOB/DMN icons gone + junk 40.5 lines showing**
- `mobile/src/store/picksStore.ts`: Added `.eq('is_smart_pick', true)` to `fetchTodayProps()`. Raw model prediction rows (e.g., bench player PRA line at 40.5, odds_type='standard') were mixed with real PP smart picks, overriding the smart pick row for the same player and hiding the GOB/DMN tag.

**UNDER button on goblin/demon lines**
- `mobile/src/components/picks/PropCard.tsx`: UNDER button only renders when `prop.odds_type === 'standard'`. PP rule: goblin/demon lines are OVER-only bets.

**No props loading**
- `mobile/src/services/supabase.ts` `getTodayDate()`: Was using UTC. At 9PM CST, UTC = next day → query returned 0 rows. Fixed to use local date.

**sync_odds_types performance**
- `sync/supabase_sync.py`: Attempted batch upsert to replace N individual UPDATE calls — failed because upsert tries INSERT on missing rows, violating NOT NULL constraints on team/opponent. Reverted to individual UPDATE by id, but added skip logic for rows where stored odds_type/edge already matches expected value.

### 6. PlayerCardModal — Player History from Supabase

`mobile/src/services/api.ts` `fetchPlayerHistory()` rewritten to query Supabase `daily_props` directly (no local FastAPI needed):
- Filters 90-day window, `ppLine ±8` to exclude junk model lines
- Deduplicates by (game_date, prop_type): prefers graded > ungraded, then standard > goblin > demon
- Computes model signals: L5/L10/season hit rates, averages, streak (consecutive HIT/MISS), trend slope (linear regression on last 10 actual values)

### 7. Expo Go Compatibility

**WorkletsError: JS 0.7.4 vs native 0.5.1**
- Removed `react-native-reanimated` from `PointsBadge.tsx` and `PickButton.tsx`, replaced with RN built-in `Animated`. Expo Go bakes its own reanimated native binary — JS version can't override it.

**OAuth redirect failing ("page address is invalid")**
- Expo Go can't handle `freepicks://` custom URI scheme after Discord OAuth. Auth succeeds server-side (user appears in Supabase) but redirect fails on device.
- Workaround: dev bypass button in `AuthScreen.tsx` → `devSignIn()` in `authStore.ts` sets a mock session/profile, skipping auth entirely.
