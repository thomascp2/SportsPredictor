# Session — 2026-03-31

## Summary
Diagnosed and fixed a systematic home-team bias in the game lines prediction engine, moved all game prediction schedules before 10 AM, added richer ML training features to all three sport feature extractors, and upgraded the dashboard's Today's Picks tab with prop/player/team checkbox filters.

---

## 1. Home Team Bias — Root Cause & Fix

### Problem
The Game Lines tab was producing an overwhelming number of home-team moneyline and spread picks. Over/unders looked fine, but directional bets were almost entirely home-favoured.

### Root Cause
`shared/game_statistical_baseline.py` — `_estimate_home_win_prob()` (line 249) applied a flat additive home advantage boost AFTER blending three signals:
- NHL: +3.5%
- NBA: +4.5%
- MLB: +2.5%

The bug was that all three input signals already encode home advantage independently:
1. **Elo** — trained on historical outcomes, which naturally reflect home court/ice edge. The default prior `gf_elo_home_prob` was 0.55 (NHL) / 0.575 (NBA), already above 50%.
2. **Market odds** — sportsbook lines are highly efficient and fully price in home field. Weighting odds 25-35% was re-importing the home edge the market already knew.
3. **Stats splits** — `gf_home_home_win_pct` (default 0.55) and `gf_away_away_win_pct` (default 0.45) directly measure actual home/away performance.

Effect: every game started with ~58-62% home win probability before any matchup signal was considered. Even strong away teams showed low probability and were rarely surfaced as SHARP/PRIME. Totals were unaffected because they're independent of the home/away bias.

### Fix
Removed the `blended += w["home_advantage"]` line. Added an explanatory comment so it's never accidentally re-added. The `home_advantage` values remain in `SPORT_PARAMS` as documented constants but are no longer applied.

**File:** `shared/game_statistical_baseline.py`

## 2. Game Prediction Schedule — Moved to Before 10 AM

### Problem
The orchestrator was generating game line predictions far too late for morning betting:
- NHL: 12:00 PM
- NBA: 1:00 PM
- MLB: 12:30 PM

### Fix
Moved all three to run before 10 AM. The team stats + Elo update (the dependency) completes well before these times:

| Sport | Team Stats | Before | After |
|-------|-----------|--------|-------|
| NHL   | 2:30 AM   | 12:00 PM | **9:00 AM** |
| NBA   | 4:30 AM   | 1:00 PM  | **9:30 AM** |
| MLB   | 7:30 AM   | 12:30 PM | **9:45 AM** |

**Note**: The orchestrator must be restarted to pick up schedule changes — `schedule.every().day.at(...)` is registered at startup, not dynamically.

**File:** `orchestrator.py` (NHL, NBA, MLB `SportConfig` classes)

## 3. ML Feature Improvements — All Three Sports

### Problem
The feature extractors stored Elo as a single diff + probability pair. Missing:
- Individual Elo ratings (absolute quality matters — two 1600-rated teams behave differently than two 1400-rated teams even at the same diff)
- Recent-form momentum (a team trending +3 goals above their season average is a much stronger pick than season stats suggest)
- L5 scoring data for totals model (current total prediction used only season averages)

### Added Features

**NHL** (`nhl/features/game_features.py`):
- `gf_home_elo` / `gf_away_elo` — raw Elo ratings from `elo_engine.get_rating()`
- `gf_home_momentum` / `gf_away_momentum` — L5 goal diff minus season goal diff
- `gf_home_l5_gf_avg` / `gf_away_l5_gf_avg` — L5 goals for average
- `gf_home_l5_ga_avg` / `gf_away_l5_ga_avg` — L5 goals against average

**NBA** (`nba/features/game_features.py`):
- `gf_home_elo` / `gf_away_elo` — raw Elo ratings
- `gf_home_momentum` / `gf_away_momentum` — L5 point diff minus season point diff
- `gf_home_l5_ppg` / `gf_away_l5_ppg` — L5 points per game
- `gf_home_l5_papg` / `gf_away_l5_papg` — L5 points allowed per game

**MLB** (`mlb/features/game_features.py`):
- `gf_home_elo` / `gf_away_elo` — raw Elo ratings

All new features default to neutral priors (Elo → 1500.0, momentum → 0.0, L5 scoring → league average) so existing predictions using default features are unaffected.

All features are written to `features_json` in `game_predictions`, which is what `train_game_models.py` reads for ML training. Momentum will be especially useful for spread and total models once enough graded samples exist.

## 4. Dashboard Checkbox Filters — Props / Players / Teams

### Problem
The Today's Picks tab had no way to narrow the picks list by specific prop types, players, or teams. The only categorical filter was the tier multiselect in the hidden "More filters" expander.

### Fix
Added a new **"Filter by Props / Players / Teams"** expander that appears after data loads (so checkboxes are populated from actual live data, not hardcoded lists). Three columns:

1. **Props** — `st.multiselect` with all unique prop types in the current result set. All selected by default; uncheck to hide unwanted props.
2. **Search player** — text input, case-insensitive substring match on `player_name`.
3. **Teams** — `st.multiselect` extracting team abbreviations from the `Matchup` column. Useful for focusing on tonight's key matchups.

Filters apply additively after `fetch_picks()` returns. If all three are at defaults (all props selected, no player search, all teams selected), the output is identical to before.

**File:** `dashboards/cloud_dashboard.py`
