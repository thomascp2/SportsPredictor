## What Was Done (Late Evening Session — Cloud Bug Fixes)

### Bugs Fixed (commit 036f8104)

1. **NBA Top Plays blank** — `tier_filter=[]` is an unhashable list, breaks `@st.cache_data`. Fixed: `tier_filter=()` in Top Plays call. Also fixed `st.multiselect` return value (list) in props tab: always `tuple(tier_filter)` at call sites.

2. **MLB player props blank** — Default tier filter shows T1+T2+T3 only. MLB had 199 T4-LEAN, 22 T3-GOOD, 7 T2-STRONG, 3 T1-ELITE. The 32 T1-T3 picks were actually there — just a filter UX issue. (T4-LEAN is visible if user adds it to tier filter.)

3. **game_time missing everywhere** — `_turso_picks_http` hardcoded `"game_time": None`. Fixed: reads `raw_row[10]` (the 11th column in the SELECT). Column order in SQL: player_name(0), team(1), opponent(2), prop_type(3), line(4), odds_type(5), prediction(6), probability(7), ai_tier(8), model_version(9), game_time(10).

4. **game_time NULL in Turso** — `sync_game_times` had run before PP lines were populated today. Manually synced: NBA (24 teams), NHL (16 teams), MLB (after ping-to-wake). A few NBA players still missing game_time due to team abbreviation mismatch — cosmetic, not worth fixing.

5. **MLB Turso wake-up** — MLB free-tier Turso sleeps. Must ping with `SELECT 1` (30s timeout) before running any sync or it fails with timeout after 3 retries.
