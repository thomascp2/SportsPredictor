## Issues Investigated & Fixed

### Bug 1 — Goblin Lines Dominating Top-20 Notification

**Root cause:** With `PROBABILITY_CAP` lifted, the statistical model returns `prob=1.0` for
trivially easy PrizePicks lines (e.g., `pts_asts 11.5` for a starting PG who averages 20+ pts
and 7+ ast). These props don't have trained ML models so they never get calibrated — they sat
at the top of every notification.

**Fix (`orchestrator.py`):**
- Added `AND probability BETWEEN 0.56 AND 0.95` filter to the top-picks SQL query
- Cuts out goblin lines at 1.0 and near-random noise at ~0.5
- Only genuine-edge picks surface

**Also fixed in same commit:**
- L5% now directional: OVER picks show OVER rate, UNDER picks show UNDER rate (`1 - l5_rate`)
- Streak `ORDER BY` now uses `ABS(f_current_streak)` so negative UNDER streaks sort correctly
- Streak display uses absolute value so UNDER streaks show correctly (was showing nothing for negative values)

### Bug 3 — Feb 25 NBA Predictions Never Generated (0 picks in DB)

**Root cause (multi-layer):**

1. **Silent exception swallowing** — `except Exception: pass` in the NBA prediction loop
   (`generate_predictions_daily_V6.py` line 402) silently discarded every per-player error.
   If something went systematically wrong at 6 AM (e.g., statistical predictor failing),
   0 predictions were generated with zero trace.

2. **Script always exited 0** — the prediction script returned code 0 whether it generated
   2000 predictions or 0. The orchestrator treats any exit 0 as success, updates
   `last_prediction_gen`, and proceeds — leaving no record of the failure.

3. **Orchestrator never verified prediction count** — after a "successful" script run,
   it never queried the DB to confirm rows were actually written.

4. **Top-picks notification silently skipped** — when no picks were found, it printed a
   local log line and returned. No Discord message. Complete silence on failure.

**Fixes (`nba/scripts/generate_predictions_daily_V6.py`):**
- `except Exception as e: pass` → `print(f"[WARN] {player} {prop} {line}: {e}", file=sys.stderr)`
- Track `skipped_errors` count, report in output and return value
- Exit code 1 when games are scheduled but 0 predictions generated
- Exit code 0 for legitimate skips (off day, already exists)

**Fixes (`orchestrator.py`):**
- After script exits 0, queries DB: `SELECT COUNT(*) FROM predictions WHERE game_date = ?`
- If count == 0: marks pipeline failed, posts Discord alert with recovery command
- `_run_script()` now writes stdout+stderr to `logs/pipeline_{SPORT}_{YYYYMMDD}.log` even on success
- `_send_discord_alert()` helper for pipeline failure/warning notifications
- Top-picks notification sends Discord warning instead of silently skipping

### Bug 4 — NHL Top-20 Picks Not Implemented

NHL uses `features_json` blob instead of individual `f_*` columns, so the existing
NBA top-picks SQL query failed silently for NHL (no matching columns).

**Fix (`orchestrator.py`):**

Refactored `run_top_picks_notification()` into:
- `_fetch_top_picks(today, direction=None)` — sport-aware fetch
  - NBA: SQL query on `f_*` columns with prob band filter
  - NHL: fetches all candidates, parses `features_json` in Python for
    `success_rate_l5`, `current_streak`, `is_home`, `games_played`;
    deduplicates to best pick per player; sorts in Python
- `_format_pick_line(i, pick)` — shared formatter, same display for both sports
- `_post_picks_to_discord(label, picks, today)` — shared post helper
